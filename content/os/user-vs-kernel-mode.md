# 🧩 Question: What is the difference between User Mode and Kernel Mode? Why can't a process read another process's memory?

## 🎯 What the interviewer is testing
- Protection Rings (Rings 0-3).
- Hardware-enforced isolation.
- Role of the Scheduler and Memory Management.

---

## 🧠 Deep Explanation

### 1. Kernel Mode (Ring 0):
- **Full Control**: The OS runs here. It can execute any CPU instruction and access ANY memory address.
- **Power**: It talks directly to the hardware (Disks, NICs).

### 2. User Mode (Ring 3):
- **Restricted**: Regular apps run here. They cannot execute privileged instructions (like `reboot` or `write-to-disk-directly`).
- **Isolation**: Every instruction is checked by the **MMU** (Memory Management Unit). If an app tries to access an address outside its Page Table, the CPU triggers a **General Protection Fault**.

### 3. Crossing the Boundary (Syscalls):
Apps shouldn't write to disk, but they need to!
- They use a **Syscall** (`open()`, `read()`).
- This causes a "Trap" to the Kernel.
- The Kernel checks permissions, performs the work in Ring 0, and returns the result to User Mode.

---

## ✅ Ideal Answer
User and Kernel modes provide the fundamental security boundary for modern computing. By restricting regular applications to a "sandbox" (User Mode) where they only see their own virtual memory, the OS prevents accidental and malicious system crashes. All privileged operations are gated through strict system calls, ensuring that the kernel remains the sole trusted arbiter of hardware resources.

## 🔄 Follow-up Questions
1. **What is a "Trap"?** (A synchronous interrupt generated by a CPU when it hits an error or a syscall.)
2. **Performance cost of mode switching?** (Very high. It requires flushing state and changing the CPU's context—one reason why "Zero-Copy" or "IO_uring" is so fast is because it minimizes these crosses.)
3. **Microkernel vs Monolithic?** (Monolithic [Linux] puts everything in Kernel; Microkernel [L4] keeps only essentials in Kernel, moving drivers to User Mode.)
