# 🧩 Question: What are Java Records? Compare them to standard POJOs and Lombok's `@Data`.

## 🎯 What the interviewer is testing
- Modern Java (14/15/16+) syntax knowledge.
- Immutability by design.
- Awareness of Data Transfer Objects (DTO) patterns.

---

## 🧠 Deep Explanation

### 1. The Secrets of `record`:
A `record` is a special class type intended to act as a **transparent data carrier**.
When you declare `record User(String name, int age) {}`:
- The compiler generates `private final` fields.
- Accessor methods (`name()`, `age()`) are generated (Note: NOT `getName()`).
- `equals()`, `hashCode()`, and `toString()` are autogenerated based on all fields.
- A **Canonical Constructor** is created.

### 2. Constraints:
- Records are implicitly `final` (cannot be extended).
- They cannot have any other instance fields except those in the header.
- They are **Immutable** (all fields are final).

### 3. Record vs Lombok:
Lombok's `@Data` is a library-level hack that modifies bytecode to add getters/setters/equals. `record` is a **language-level** feature with built-in semantics, making it safer and more efficient for the JVM to optimize.

---

## ✅ Ideal Answer
Java Records are the modern answer to boilerplate-heavy POJOs. They emphasize data ownership and immutability, making code more readable and thread-safe by default. Since they are a language feature, they avoid the "black magic" of Lombok while providing better integration with modern Java features like Pattern Matching.

---

## 💻 Java Code
```java
// Definition
public record Point(int x, int y) {}

// Usage
Point p = new Point(10, 20);
System.out.println(p.x()); // 10
System.out.println(p);   // Point[x=10, y=20]
```

---

## 🔄 Follow-up Questions
1. **Can a record implement an interface?** (Yes.)
2. **Can you customize the constructor?** (Yes, using a "Compact Constructor.")
3. **Difference between `p.x()` and `p.getX()`?** (Records use the name as the accessor; standard beans use the `get` prefix. This can break some legacy libraries that rely strictly on JavaBeans specs.)
